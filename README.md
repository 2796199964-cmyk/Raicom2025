# 2025 RAICOM 平安城市

本项目整理 2025 睿抗机器人开发者大赛平安城市赛题相关资料与算法代码，主要面向第一次参加 RAICOM 平安城市赛题的同学。仓库保留了不同视觉任务的样例数据、初版算法和处理结果，方便同学提前了解任务形式、输入输出要求和常见解题思路。

平安城市赛题通常围绕机器人视觉感知展开，需要参赛者根据图像或视频信息识别道路环境、人员目标、建筑状态和交通设施。相比单纯调用模型，比赛更看重对场景特征的理解、图像预处理、规则判断和结果表达。

## 仓库内容

| 目录 | 内容 | 说明 |
| --- | --- | --- |
| `人偶分类/` | 人偶图片、模板匹配结果、`标记职业人数.py` | 使用 SIFT 特征匹配识别职业人员目标，并在图像中标记匹配结果 |
| `垃圾桶/` | 垃圾桶场景图片 | 用于垃圾桶识别或场景目标检测的样例数据 |
| `楼宇/` | 楼宇场景图片 | 用于楼宇、楼层、火灾等视觉识别任务的样例数据 |
| `红绿灯/` | 红绿灯图片、`traffic lights recognition.py` | 使用 HSV 颜色分割与位置规则判断红灯、黄灯、绿灯 |

## 任务说明

### 人偶分类

`人偶分类/标记职业人数.py` 使用 SIFT 提取模板图像和目标图像的特征点，再通过 FLANN 匹配器寻找相似区域。程序会根据单应性矩阵估计模板在目标图中的位置，并通过角度、面积和凸四边形等条件过滤误匹配。

运行后，结果会保存到 `人偶分类/match_results/`。代码中的 `len(valid_matches)` 对应当前图像中识别到的职业人员数量。

### 红绿灯识别

`红绿灯/traffic lights recognition.py` 将图像转换到 HSV 颜色空间，分别提取红色、黄色、绿色区域。程序会根据灯的排列方向、亮灯位置和颜色面积判断当前信号灯状态，并把“红灯”“黄灯”“绿灯”等结果写回图片。

运行后，结果会保存到 `红绿灯/hld/processed_results/`。

### 垃圾桶与楼宇

`垃圾桶/` 和 `楼宇/` 目前主要保存样例图片，可作为后续目标检测、颜色识别、OCR 或规则判断算法的输入数据。建议后续把对应算法单独整理为 `.py` 文件，并在文档中补充运行命令和输出格式。

## 环境依赖

建议使用 Python 3.9 或以上版本。安装依赖：

```bash
pip install -r requirements.txt
```

如果只运行红绿灯识别，通常安装 `opencv-python`、`numpy`、`Pillow` 即可。人偶分类脚本用到了 `matplotlib`，并依赖 OpenCV 的 SIFT 功能；如果本地 OpenCV 版本不支持 `cv2.SIFT_create()`，请安装 `opencv-contrib-python`。

## 运行示例

进入对应目录后运行脚本。

```bash
cd 人偶分类
python 标记职业人数.py
```

```bash
cd 红绿灯
python "traffic lights recognition.py"
```

Windows PowerShell 中路径包含空格时需要加引号，例如：

```powershell
python ".\traffic lights recognition.py"
```

## 适合新同学怎么参考

第一次参加比赛时，可以先从已有图片和代码理解每个任务的判断目标，而不是一开始就追求复杂模型。红绿灯识别适合理解 HSV 颜色分割和规则判断；人偶分类适合理解模板匹配、特征点、误匹配过滤和结果可视化。

如果想继续完善项目，建议优先做三件事：把每个任务的输入输出格式写清楚；把路径改成命令行参数；把运行结果和错误情况记录到文档中。这样后续同学复现代码时会更容易。

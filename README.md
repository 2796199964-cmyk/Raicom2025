## 📖 项目简介

本项目主要面向**第一次参加 RAICOM 平安城市赛题的同学**。仓库系统性地整理了赛题的样例数据、初版算法和处理结果，旨在帮助新手提前了解任务形式、输入输出要求以及常见的解题思路。

> 💡 **赛题核心理念**：平安城市赛题围绕机器人视觉感知展开。相比单纯调用深度学习模型，比赛**更看重对场景特征的理解、图像预处理、传统 CV 规则判断和结果的精准表达**。

## 👥 团队分工

- 👤 **人偶分类**：由本人（仓库 Owner）负责核心算法设计、代码编写与调试。

## 🎯 赛题任务拆解

本仓库涵盖了平安城市赛题的四个核心视觉感知模块：

### 1. 🕵️ 人偶分类 (核心算法)
- **任务目标**：在复杂场景中识别特定的“职业人员”目标，并准确统计人数。
- **技术路线**：摒弃了重度依赖算力的深度学习，采用传统 CV 的 **SIFT 特征提取 + FLANN 匹配器**。
- **核心逻辑**：
  1. 提取模板图像与目标图像的 SIFT 特征点。
  2. 通过单应性矩阵（Homography）估计模板在目标图中的透视变换位置。
  3. 引入**角度、面积和凸四边形**等几何条件，严格过滤误匹配。
- **输出结果**：`len(valid_matches)` 即为识别到的职业人员数量，可视化结果保存至 `人偶分类/match_results/`。
- **核心文件**：`人偶分类/标记职业人数.py`

### 2. 🚦 红绿灯识别
- **任务目标**：判断当前交通信号灯的状态（红灯、黄灯、绿灯）。
- **技术路线**：基于 **HSV 颜色空间分割** 与 **位置规则判断**。
- **核心逻辑**：将图像转换至 HSV 空间，分别提取红、黄、绿高亮区域；结合灯的排列方向（横排/竖排）、亮灯相对位置和颜色面积占比，综合推断当前信号灯状态。
- **输出结果**：将“红灯/黄灯/绿灯”等状态文本渲染至图片，保存至 `红绿灯/hld/processed_results/`。
- **核心文件**：`红绿灯/traffic lights recognition.py`

### 3. 🏢 楼宇状态监测
- **任务目标**：识别楼宇名称、定位火灾楼层等建筑状态。
- **技术路线**：参考 2024 届 RAICOM 平安城市算法（OCR + 区域网格划分）。
- **当前状态**：`楼宇/` 目录目前主要保存样例图片，作为后续目标检测、OCR 或规则判断算法的输入数据。

### 4. ♻️ 垃圾桶识别
- **任务目标**：识别垃圾桶场景目标或进行垃圾分类判断。
- **当前状态**：`垃圾桶/` 目录目前仅上传了场景样例数据，暂未上传对应算法方案。

### 5. 🛣️ 巡线任务
- **任务目标**：基于官方提供的视频数据集或实车摄像头进行巡线处理。**核心规则**：必须精准识别车道的**两侧边缘线**（严禁仅识别车道中心线），最好输出对应的车辆转向角度值（Steering Angle）。
- **技术路线**：算法逻辑可参考 [Raicom2024 - P1 轨道线识别](https://github.com/2796199964-cmyk/Raicom2024/tree/main/src)（主要涉及 ROI 掩膜提取、Canny 边缘检测、霍夫直线拟合与转向角度计算）。
- **当前状态**：`巡线/` 目录目前仅上传了巡线视频，未上传具体算法代码。
